# NiumaScene 核心场景制作指南

## 目标
核心场景用于承载跨场景常驻服务，解决切换 RPG、建筑、MiniGame、战斗测试场景时的状态丢失问题。

一句话原则：

```text
核心场景放全局服务，业务场景放内容和局部表现。
```

从 NiumaUI 2.1.0 起，正式 UI 主线是 **UI Toolkit**。核心场景文档不再推荐旧 UGUI `UIManager / DefaultViewFactory / DialogueWindowView / DialogueWindowBinding`。旧场景如果还有这些 Missing Script，应删除后改接 `UIToolkitUIManager + UIToolkitViewFactory + ToolkitReceiver + ToolkitBindingProvider`。

## 推荐层级
推荐一个模块或功能集绑定一个物体。不要把所有脚本都堆在同一个 GameObject 上。

```text
CoreScene
└── BootstrapRoot
    ├── SceneRoot
    │   ├── LoadingRoot
    │   ├── SceneAudioRoot
    │   └── CheckpointRequester
    ├── AudioRoot
    ├── UIRoot
    │   ├── EventSystem
    │   ├── UIToolkitRoot
    │   │   ├── UIDocument_HUD
    │   │   ├── UIDocument_Prompt
    │   │   ├── UIDocument_Dialogue
    │   │   ├── UIDocument_Menu
    │   │   ├── UIDocument_Popup
    │   │   ├── UIDocument_Loading
    │   │   └── UIDocument_Debug
    │   ├── UIManager
    │   ├── UIBridges
    │   └── BindingProviders
    ├── SaveRoot
    │   └── SaveAdapters
    ├── GameplayServicesRoot
    └── MiniGameSessionRoot
```

说明：`UIRoot/UIManager` 是物体名，不是旧 UGUI `UIManager` 脚本。这个物体上挂 `UIToolkitUIManager` 和 `UIToolkitViewFactory`。

## Inspector 配置阅读规则
配置表中的“可留空”含义：

- `可以`：留空不会阻断主流程，脚本会自动查找或跳过对应表现。
- `不建议`：留空通常还能跑，但正式场景容易找错对象或缺少表现。
- `不可以`：正式流程必须填写，否则功能无法执行。

## 最小可运行核心场景
只想先解决返回栈和跨场景常驻时，最小层级如下：

```text
BootstrapRoot
└── SceneRoot
```

### BootstrapRoot
挂载脚本：`NiumaGameBootstrapper`

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Bootstrap Root` | 拖 `BootstrapRoot` 自己 | 可以 | 为空时使用当前 GameObject |
| `Dont Destroy On Load` | 正式场景开启 | 不建议关闭 | 关闭后切场景会卸载核心服务，返回栈丢失 |
| `Destroy Duplicate Bootstrap` | 正式场景开启 | 不建议关闭 | 重复进入 CoreScene 时可能出现两套全局服务 |
| `First Scene Name` | 填第一个业务场景名，如 `RPG_Village` | 可以 | 为空时启动后停留在 CoreScene |
| `Load First Scene On Start` | 需要自动进游戏时开启 | 可以 | 关闭后要按钮或脚本手动加载首场景 |
| `Clear Return Stack Before First Scene` | 正式启动流程开启 | 可以 | 关闭后旧返回栈可能残留 |
| `Show Loading UI On First Scene` | 有 Loading View 时开启 | 可以 | 关闭后首场景加载不显示 Loading |
| `Scene Controller` | 拖 `SceneRoot` 上的 `NiumaSceneController` | 不建议 | 自动查找失败时无法进入首场景 |

### SceneRoot
挂载脚本：`NiumaSceneController`

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Fallback Scene Name` | 安全兜底场景名，如 `RPG_Village` / `MainMenu` | 可以 | 加载失败时只记录错误，不自动回兜底场景 |
| `Max Return Context Depth` | 建议 8 | 不可以 | 太小会导致嵌套返回不足 |
| `Default Return Overflow Policy` | 建议 `RejectNew` | 不可以 | 策略不当可能丢返回链 |
| `Freeze Input During Load By Default` | 正式跨场景加载建议开启 | 可以 | 加载期间玩家可能继续移动/交互 |
| `Show Loading UI By Default` | 有 Loading View 时开启 | 可以 | 默认不显示 Loading |
| `Checkpoint Requester Provider` | 拖 `SceneRoot/CheckpointRequester` 上的 `NiumaSceneSaveCheckpointRequester` | 可以 | 勾选检查点保存时不会真正写档 |
| `Auto Find Checkpoint Requester` | 测试可开，正式建议手动绑定 | 可以 | 关闭且未绑定时检查点保存不可用 |

全项目只保留一个 `NiumaSceneController`。RPG、建筑、MiniGame 业务场景不要重复挂。

## UI Toolkit 核心配置
### UIRoot
推荐放置：

```text
UIRoot
├── EventSystem
├── UIToolkitRoot
│   ├── UIDocument_HUD
│   ├── UIDocument_Prompt
│   ├── UIDocument_Dialogue
│   ├── UIDocument_Menu
│   ├── UIDocument_Popup
│   ├── UIDocument_Loading
│   └── UIDocument_Debug
├── UIManager
│   ├── UIToolkitUIManager
│   └── UIToolkitViewFactory
├── UIBridges
│   ├── SceneLoadingToolkitBridge
│   ├── InteractionPromptToolkitSink
│   ├── NiumaGalToolkitDialogueViewBridge
│   └── 各业务模块 ToolkitReceiver
└── BindingProviders
    ├── DialogueToolkitBindingProvider
    ├── AmbientNarrativeToolkitBindingProvider
    ├── InteractionPromptToolkitBindingProvider
    ├── ToolkitLoadingBindingProvider 或自定义 Loading Provider
    └── 各业务模块 ToolkitBindingProvider
```

### UIToolkitUIManager
挂载位置：`CoreScene/BootstrapRoot/UIRoot/UIManager`

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `View Factory` | 拖同物体上的 `UIToolkitViewFactory` | 不建议 | 无法打开任何 Toolkit View |
| `Input Blocker Provider` | 拖 `TPCGameplayInputBlocker`，通常在玩家 `PlayerRoot/UIBridge` 上 | 可以 | 菜单/对话/Loading 打开时不会冻结玩家控制 |
| `Toast / Confirm / Loading ViewId` | 填注册表中的通用 ViewId | 可以 | 对应通用弹窗不可用 |
| `Enable Keyboard Back` | 需要 ESC 返回时开启 | 可以 | 关闭后需要外部输入系统调用返回 |
| `Drive Tick In Update` | 没有统一模块 Tick 时开启 | 可以 | 外部统一 Tick 时应关闭，避免重复驱动 |

`Input Blocker Provider` 不是拖 `SceneLoadingStateBridge` 的输入目标。它是 UI 打开菜单、对话、Loading 时冻结玩法输入的桥接，推荐拖 `TPCGameplayInputBlocker`。

### UIToolkitViewFactory
挂载位置：和 `UIToolkitUIManager` 同物体。

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Registry` | 拖 `UIToolkitViewRegistrySO` | 不建议 | 无法按 ViewId 创建 View |
| `Layer Roots` | 每层拖对应 `UIDocument` 的 Root | 不建议 | 对应层无法显示 View |
| `Binding Provider Behaviours` | 拖 `BindingProviders` 下的各模块 Provider | 可以 | 未匹配时使用空 Binding，窗口可能无内容 |
| `Log Warnings` | 建议开启 | 可以 | ViewId、Layer、Provider 配错时不提示 |

### UIToolkitViewRegistrySO
创建方式：Project 右键 `Create / NiumaUI / Toolkit View Registry`。

常用 ViewId 建议：

| ViewId | LayerId | BindingProviderId | InputPolicy | 用途 |
| --- | --- | --- | --- | --- |
| `Loading` | `Loading` | `Loading` 或自定义 | `BlockGameplayInput` | 场景加载遮罩 |
| `DialogueWindow` | `Dialogue` | `DialogueWindow` | `BlockGameplayInput` | Gal 对话窗口 |
| `AmbientNarrative` | `HUD` / `Prompt` | `AmbientNarrative` | `None` | 环境叙事气泡/字幕 |
| `InteractionPrompt` | `Prompt` | `InteractionPrompt` | `None` | 交互提示 |
| `InventoryPanel` | `Menu` | `InventoryPanel` | `BlockGameplayInput` | 背包面板 |
| `SavePanel` | `Menu` | `SavePanel` | `BlockGameplayInput` | 存档面板 |
| `QuestTracker` | `HUD` | `QuestTracker` | `None` | 任务追踪 |
| `MiniGameStartPanel` | `Menu` | `MiniGameStartPanel` | `BlockGameplayInput` | 小游戏开始/大厅 |
| `MiniGamePlayPanel` | `Menu` | `MiniGamePlayPanel` | `BlockGameplayInput` | 小游戏进行中 |

## Loading 接入
### 推荐：SceneLoadingToolkitBridge
挂载位置：`CoreScene/BootstrapRoot/UIRoot/UIBridges/SceneLoadingToolkitBridge`

用途：把 `SceneLoadingSnapshot` 转成 NiumaUI Toolkit Loading View。

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `UI Manager` | 拖 `UIRoot/UIManager` 上的 `UIToolkitUIManager` | 可以 | 自动查找仍失败时不显示 Loading |
| `Auto Find UI Manager` | 测试可开，正式建议手动绑定后关闭 | 可以 | 关闭且未绑定时 Loading 不显示 |
| `Loading Text` | 如 `加载中` | 可以 | 使用脚本默认文案 |
| `Activating Text` | 如 `正在进入` | 可以 | 使用脚本默认文案 |
| `Completed Text` | 如 `加载完成` | 可以 | 使用脚本默认文案 |
| `Failed Text` | 如 `加载失败` | 可以 | 使用脚本默认文案 |
| `Hide When Loading Ends` | 建议开启 | 可以 | 关闭后加载结束不主动隐藏 Loading |
| `Use Snapshot Input Block Flag` | 建议开启 | 可以 | 关闭后 Loading View 始终按阻塞处理 |
| `Log Warnings` | 建议开启 | 可以 | 缺 UIManager 时不提示 |

绑定步骤：

1. 在 `UIRoot/UIBridges` 下创建 `SceneLoadingToolkitBridge`。
2. 挂 `SceneLoadingToolkitBridge`。
3. `UI Manager` 拖 `UIRoot/UIManager` 上的 `UIToolkitUIManager`。
4. 在 `UIToolkitViewRegistrySO` 中注册 `Loading` View。
5. 在 `UIToolkitViewFactory.Binding Provider Behaviours` 中拖入 Loading 的 BindingProvider。
6. 选中 `SceneRoot/LoadingRoot/SceneLoadingStateBridge`。
7. 把 `SceneLoadingToolkitBridge` 拖到 `Loading Receiver Provider`。

### Legacy：SceneLoadingPanelBridge
`SceneLoadingPanelBridge` 是旧 UGUI/TMP Loading 面板桥接。2.1.0 正式场景不再推荐使用。旧测试场景如果暂时保留它，不要同时绑定 `SceneLoadingToolkitBridge`，否则两个 Loading UI 会同时响应。

## SceneLoadingStateBridge
挂载位置：`CoreScene/BootstrapRoot/SceneRoot/LoadingRoot`

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Scene Controller` | 拖 `SceneRoot` 上的 `NiumaSceneController` | 不建议 | 自动查找失败时无法读取加载状态 |
| `Loading Receiver Provider` | 正式拖 `SceneLoadingToolkitBridge`；旧测试可拖 `SceneLoadingPanelBridge` | 可以 | 留空时只冻结/解冻输入，不显示 Loading UI |
| `Input Block Target Providers` | 玩家/交互常驻核心场景时手动拖 `TPCSceneInputBlockTarget`、`InteractSceneInputBlockTarget` | 可以 | 留空且关闭自动查找时不冻结对应输入 |
| `Auto Find Input Block Targets` | 玩家/交互在业务场景时开启 | 可以 | 关闭且未手动绑定时不冻结业务输入 |
| `Input Block Reason` | 默认 `SceneLoading` | 不建议为空 | 为空时难以按来源释放阻塞 |
| `Unblock When Loading Ends` | 建议开启 | 可以 | 关闭后加载结束不会自动释放本桥接加的输入阻塞 |

## 检查点保存
### NiumaSceneSaveCheckpointRequester
挂载位置：`CoreScene/BootstrapRoot/SceneRoot/CheckpointRequester`

正确链路：

```text
NiumaSceneController.Checkpoint Requester Provider
    -> SceneRoot/CheckpointRequester/NiumaSceneSaveCheckpointRequester
        -> SaveRoot/NiumaSaveController
            -> 各模块 SaveAdapter 导出 Section
```

业务场景只负责在入口按钮、传送点或对话跳转中勾选 `Request Checkpoint Save`。

## 出生点与按钮跳转
### SceneSpawnPoint
挂载位置：业务场景中的 `SpawnPoint_xxx` 物体。

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Spawn Point Id` | 稳定 ID，如 `main_default`、`npc_minigame_return` | 不可以 | 返回/传送找不到正确落点 |
| `Is Default` | 每个场景建议只勾一个默认点 | 可以 | 未指定点且无默认点时恢复位置可能失败 |
| `Draw Gizmos` | 建议开启 | 可以 | Scene 视图看不到落点方向 |

### SceneButtonAction
挂载位置：业务场景 UI 按钮、门、传送点或对话入口物体。

| 字段 | 怎么填 | 可留空 | 不填/关闭会怎样 |
| --- | --- | --- | --- |
| `Scene Controller` | 拖核心场景 `SceneRoot` 上的 `NiumaSceneController` | 不建议 | 自动查找失败时按钮无效 |
| `Target Scene Name` | 填 Build Settings 中场景名 | 加载场景时不可以 | `LoadConfiguredScene()` 不执行 |
| `Purpose` | 选 `MiniGame` / `EnterBuilding` / `Teleport` 等 | 可以 | `None` 按普通切换处理 |
| `Push Return Context` | 需要返回时开启 | 可以 | 关闭后 `ReturnToPreviousScene()` 找不到来源 |
| `Show Loading UI` | 跨场景加载建议开启 | 可以 | 不显示 Loading View |

## 对话选项触发场景跳转
对话选项进入小游戏或建筑时，不建议直接在按钮 UI 事件中写跳转。推荐：

1. `DialogueAsset` 的选项填写稳定 `ChoiceId`，如 `enter_draw_guess`。
2. Gal / Story / Interact 桥接层根据 `ChoiceId` 触发场景跳转。
3. 场景跳转仍走 `NiumaSceneController` 或封装好的 `SceneButtonAction`。

这样不会绕过 Gal 的选择记录、任务推进和存档事实。

## 业务场景还需要放什么
- RPG 场景：玩家、NPC、交互物、任务触发物、`SceneSpawnPoint`、局部 UI 入口按钮。
- MiniGame 场景：推荐 `NiumaMiniGameController` 放 `MiniGameSessionRoot` 常驻；UI 走 `MiniGameStartPanel`、`MiniGamePlayPanel`。
- 建筑 / 室内场景：室内内容、入口/出口 `SceneSpawnPoint`、门或按钮跳转脚本。

## 常见错误
### ReturnContextMissing
入口没有开启 `PushReturnContext`，或 `NiumaSceneController` 跟随业务场景被卸载。

### Loading 不显示
- `SceneLoadingStateBridge.Loading Receiver Provider` 没拖 `SceneLoadingToolkitBridge`。
- `SceneLoadingToolkitBridge.UI Manager` 没绑定 `UIToolkitUIManager`。
- `UIToolkitViewRegistrySO` 没注册 `Loading` View。
- `UIToolkitViewFactory.Layer Roots` 没配置 Loading 层的 `UIDocument`。

### UI 或音频重复
多个业务场景重复放 `UIToolkitUIManager`、`NiumaAudioController` 或 `EventSystem`。正式流程应把全局 UI、音频、存档放核心场景。

## 推荐检查清单
- `CoreScene` 在 Build Settings 第 0 个。
- `BootstrapRoot` 挂 `NiumaGameBootstrapper`，并开启 `Dont Destroy On Load`。
- `SceneRoot` 挂 `NiumaSceneController`。
- `UIRoot/UIManager` 只挂 `UIToolkitUIManager` 与 `UIToolkitViewFactory`。
- `UIToolkitViewRegistrySO` 注册 Loading、DialogueWindow、InteractionPrompt 等基础 View。
- `SceneLoadingStateBridge.Loading Receiver Provider` 绑定 `SceneLoadingToolkitBridge`。
- 全局音频只有一个 `NiumaAudioController`。
- 全局存档只有一个 `NiumaSaveController`，SaveAdapter 放核心场景。
- 正式按钮走 `SceneButtonAction` 或业务桥接，不直接调用 `SceneManager.LoadScene`。
